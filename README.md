# Тестовое задание для ML-инженера проекта Nastavnik

**КОММЕНТАРИИ**
- В условиях написано, что target = success_rate > 0.75. При этом success_rate - это один из предикторов. Если данных для обучения будет побольше, предсказание сведется к сравнению success_rate с 0.75.
- Не совсем понял условие:  "learning_curve: (успешных попыток на последних 3) / 3, если < 3 попыток то макс 1". Что здесь значит "макс 1"? В своей версии просто взял среднее за последние 3 попытки или если меньше 3, то за все.

1. **Описание подхода:**
**Подготовка данных для обучения**. Все признаки находятся в диапазоне [0, 1], кроме num_attempts, который может быть гораздо больше 1. Можно было бы использовать обычную стандартизацию (среднее 0, ст. отклонение 1), но на будущее для интерпретируемости оставил все признаки (кроме num_attempts) как есть. num_attempts тоже привел к этому диапазону. Дальше использовал лог. регрессию с параметрами по умолчанию, в т.ч. регуляризацию.

**Использовал все предложенные упрощения**, т.к. объем данных маленький: 
- Не использовал PyTorch для обучения, только sklearn. 
- Не использовал кросс-валидацию или разделение на train/validation/test.
- Модель сохранил в файле с помощью pickle, не использовал dependency. Загрузил как переменную
- Примитивная проверка состояния сервиса (есть модели или нет).

2. **Как запустить:**
   ```bash
   python3 train.py  # обучение
   uvicorn app:app --reload  # сервис
   ```
